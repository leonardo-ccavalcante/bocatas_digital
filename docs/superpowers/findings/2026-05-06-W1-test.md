### F-W1T-07: Catch blocks without narrowing
**Severity:** MEDIUM (catalog only — no individual finding per file)
**Audit:** H
**File(s):** ~25 occurrences across server/, client/src/ (uncounted in detail; exact file:lines deferred to QA-4 sub-audit)
**Why:** `catch (err: unknown)` or `catch (e)` without `instanceof Error` narrowing forces TypeScript to widen access to `(err as Error).message` → potential runtime crash if non-Error thrown.
**Fix sketch:** QA-4 subagent: `instanceof Error ? err.message : String(err)` pattern, applied per file.
**Verification:** Final `rg 'err\.message\|error\.message' --type ts --type tsx` only inside `instanceof Error` branches.
**Risk if ignored:** Crash on unusual throwables (non-Error rejection from third-party libs).
