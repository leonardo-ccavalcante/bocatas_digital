## Dimension F — Performance

### F-W2F-01: Recharts vendor chunk exceeds 100KB gzip route budget
**Severity:** CRITICAL
**Audit:** F
**File(s):** `package.json` recharts dependency; lazy-loaded by `client/src/pages/Dashboard.tsx` and consumers
**Why:** vendor-charts chunk is 103.62 KB gz. CLAUDE.md §3 says "no heavy chart libraries; bundle <300KB total" but Phase 6 top-1% target is 100KB per route chunk. Already noted as F-04 in `2026-05-06-W1-sec.md`.
**Fix sketch:** Replace recharts with a lighter alternative — `visx` (modular), `chartist`, or hand-rolled SVG sparkline. For Bocatas dashboard's small set of charts (KPI cards + 4-week trend), a 30-line custom SVG component is sufficient and adds ~3KB instead of 100KB.
**Verification:** `pnpm build && grep "vendor-charts\|index" /tmp/build-out.txt` — target chart chunk <30KB gz.
**Risk if ignored:** LCP regression on /inicio (admin dashboard); fails Phase 6 §B.1 budget; secondary risk: Lighthouse mobile score below 99.

**Severity:** HIGH

**File(s):** dist/public/assets/index-*.js + vendor-charts-*.js
**Why:** main 94.56 KB + dashboard charts 103.62 KB = 198.18 KB on first visit to `/inicio` if dashboard is initial route. Top-1% target is ≤180 KB gz.

**Risk if ignored:** First-paint regression specifically on the admin/voluntario landing page. Direct impact on the funder-facing dashboard demo.

## Dimension G — Accessibility

### F-W2G-01: 14 `<img>` elements without `alt` attribute
**Severity:** HIGH
**Audit:** G
**File(s):** 14 occurrences (verified independently of the W2-fe agent's spot-check):
- `client/src/components/AnnouncementImageUploader.tsx:106`
- `client/src/components/PhotoUploadInput.tsx:202`
- `client/src/components/layout/AppShell.tsx:163, 175, 270, 305` (4 — likely brand/logo)
- `client/src/features/checkin/components/ManualSearchModal.tsx:85`
- `client/src/features/families/components/IdentityVerifier.tsx:92`
- `client/src/features/persons/components/DocumentCaptureInline.tsx:200`
- `client/src/features/persons/components/DocumentCaptureModal.tsx:204`
- `client/src/features/persons/components/RegistrationWizard/steps/Step6Foto.tsx:33`
- `client/src/features/programs/components/DocumentPhotoCapture.tsx:191`
- `client/src/pages/Login.tsx:43`
- `client/src/pages/NovedadDetalle.tsx:74`
**Why:** WCAG 2.1 AA requires every `<img>` to have an `alt` (informative) or `alt=""` (decorative). CLAUDE.md guard-rail: WCAG 2.1 AA non-negotiable.
**Fix sketch:** QA-6 — triage each by content. Decorative (logos, repeated brand) → `alt=""`. Informative (person photo, document, screenshot) → meaningful alt text in Spanish. **NOT batch-applied** (covers SAT §J Assumption #11).
**Verification:** `grep -rnE '<img\b' client/src/ --include='*.tsx' | grep -v 'alt=' | wc -l` → 0.
**Risk if ignored:** Screen-reader users (low-vision elderly beneficiaries) cannot identify images; failed WCAG 2.1 AA → CLAUDE.md guard-rail violation.

### F-W2G-02: Decorative profile image — alt intent verification
**Severity:** MEDIUM
**Audit:** G
**File(s):** `client/src/features/checkin/components/ManualSearchModal.tsx:85-89`
**Why:** Profile image rendered with empty alt; in a list context where each row also shows the name, decorative is correct. But screen-reader users searching by face cannot get a hint from photo presence.
**Fix sketch:** Convert to `alt={person.nombre}` for screen-reader parity with sighted scan. Tradeoff documented.
**Verification:** Screen-reader announces person name twice on row focus → acceptable.
**Risk if ignored:** Accessibility parity gap on the manual-fallback flow (CLAUDE.md guard-rail: low-literacy + multi-language).

### F-W2G-03: Clickable div without role
**Severity:** LOW
**Audit:** G
**File(s):** `client/src/components/ExportFamiliesModal.tsx:86`
**Why:** `<div className="cursor-pointer" onClick={...}>` — keyboard cannot focus, screen reader announces as text, not actionable.
**Fix sketch:** Convert to `<button type="button" role="radio" aria-checked={selected}>` — semantic + a11y.
**Verification:** Tab into modal → reaches mode selectors → Space selects.
**Risk if ignored:** Power-user export flow inaccessible via keyboard; small population, but real cost.
